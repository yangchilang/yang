
# AI塔罗牌解读应用 - 产品需求文档

## 1. 产品概述

AI塔罗牌解读应用，为用户提供艺术风格的塔罗解读体验。用户可以选择塔罗牌并获取AI生成的个性化解读，结合传统塔罗牌的神秘美学。**新增用户系统，支持保存和管理解读历史。**

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|---------|---------|
| 访客用户 | 无需注册 | 使用塔罗解读功能、下载解读图片 |
| 注册用户 | 邮箱注册 | 使用塔罗解读功能、下载解读图片、保存解读历史、管理个人解读 |

### 2.2 功能模块

#### 2.2.1 用户认证模块
- **用户注册**: 邮箱、密码（加密存储）
- **用户登录**: JWT Token认证
- **会话管理**: Token刷新、自动登出

#### 2.2.2 解读历史模块
- **保存解读**: 记录每次解读的塔罗牌、正逆位、AI解读结果、创建时间
- **查看历史**: 按时间倒序展示个人解读记录
- **删除记录**: 用户可删除自己的解读历史
- **查看详情**: 查看某次解读的完整信息

#### 2.2.3 原有塔罗解读模块（保持不变）
- 塔罗牌选择器、正逆位设置、开始解读按钮
- AI解读结果展示、精美视觉呈现、生成图片、下载选项

### 2.3 页面结构
| 页面名称 | 功能描述 | 访问权限 |
|---------|---------|---------|
| 首页 | 塔罗牌解读主界面 | 公开 |
| 登录页 | 用户登录表单 | 公开 |
| 注册页 | 用户注册表单 | 公开 |
| 历史记录页 | 查看个人解读历史 | 需登录 |
| 解读详情页 | 查看某次解读的详细信息 | 需登录 |

## 3. 核心流程

### 3.1 用户认证流程
```
注册 → 邮箱验证 → 登录 → JWT Token → 访问受保护资源
```

### 3.2 解读历史流程
```
用户登录 → 创建解读 → 保存到历史 → 查看历史列表 → 查看详情/删除
```

### 3.3 解读流程（增强版）
```
选择塔罗牌 → 设置正逆位 → 点击开始解读 → 查看AI解读结果 
    ↓
[可选] 登录/注册 → 保存到历史 → 下载保存
```

## 5. 技术架构

### 5.1 技术栈
- **前端**: React@18 + TypeScript + Vite（现有）
- **后端**: Express.js + TypeScript
- **数据库**: SQLite（轻量级，适合快速原型）
- **认证**: JWT Token
- **密码加密**: bcrypt
- **状态管理**: React useState / Zustand

### 5.2 API设计

#### 认证相关
| 端点 | 方法 | 描述 | 权限 |
|------|------|------|------|
| /api/auth/register | POST | 用户注册 | 公开 |
| /api/auth/login | POST | 用户登录 | 公开 |
| /api/auth/me | GET | 获取当前用户信息 | 需要Token |

#### 解读历史相关
| 端点 | 方法 | 描述 | 权限 |
|------|------|------|------|
| /api/readings | GET | 获取用户解读历史列表 | 需要Token |
| /api/readings | POST | 保存新的解读 | 需要Token |
| /api/readings/:id | GET | 获取某次解读详情 | 需要Token |
| /api/readings/:id | DELETE | 删除解读记录 | 需要Token |

### 5.3 数据模型

#### User 用户表
```typescript
interface User {
  id: number;
  email: string;
  password: string; // bcrypt加密
  created_at: Date;
  updated_at: Date;
}
```

#### Reading 解读记录表
```typescript
interface Reading {
  id: number;
  user_id: number;
  cards: string; // JSON字符串：SelectedCard[]
  interpretation: string;
  user_context: string;
  created_at: Date;
}
```

## 6. 用户体验优化

### 6.1 未登录用户
- 可以使用全部塔罗解读功能
- 解读完成后提示登录以保存历史
- 登录/注册入口始终可见

### 6.2 已登录用户
- 自动保存每次解读到历史
- 顶部显示用户信息
- 可以查看和管理历史记录
- 登出功能

### 6.3 错误处理
- 注册：邮箱已存在、密码强度不足
- 登录：邮箱不存在、密码错误
- 历史操作：无权限、数据不存在

## 7. 安全考虑

- 密码使用 bcrypt 加密存储
- JWT Token设置合理的过期时间
- 所有密码重置操作需要验证邮箱
- 敏感操作需要用户确认
- 防止SQL注入和XSS攻击
