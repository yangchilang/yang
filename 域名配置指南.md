## 需要完成的配置步骤

### 第一步：配置 DNS 记录

1. 打开 Cloudflare DNS 配置页面：
   ```
   https://dash.cloudflare.com/5fa2e567e14198137d715a6f1e7ca114/tarot-yue.cn/dns/records
   ```

2. 点击 **"Add record"** 按钮添加两条记录：

   **记录 1 - A 记录（主域名）：**
   - Type: `A`
   - Name: `@`
   - IPv4 address: `104.18.14.24`
   - TTL: `Auto`
   - Proxy status: `Proxied`（橙色云朵图标）

   **记录 2 - A 记录（www 子域名）：**
   - Type: `A`
   - Name: `www`
   - IPv4 address: `104.18.14.24`
   - TTL: `Auto`
   - Proxy status: `Proxied`（橙色云朵图标）

3. 点击 **"Save"** 保存记录

---

### 第二步：在 DNS 中添加 CNAME 记录指向前端（更简单的方法）

不需要进入 Workers 设置，直接在 DNS 页面添加 CNAME 记录：

1. 打开 Cloudflare DNS 配置页面：
   ```
   https://dash.cloudflare.com/5fa2e567e14198137d715a6f1e7ca114/tarot-yue.cn/dns/records
   ```

2. 点击 **"Add record"**

3. 添加 CNAME 记录：
   - Type: `CNAME`
   - Name: `@`
   - Target: `ai-tarot-frontend.1547859360.workers.dev`
   - TTL: `Auto`
   - Proxy status: `Proxied`（橙色云朵图标）

4. 再添加一条 www 的 CNAME 记录：
   - Type: `CNAME`
   - Name: `www`
   - Target: `ai-tarot-frontend.1547859360.workers.dev`
   - TTL: `Auto`
   - Proxy status: `Proxied`（橙色云朵图标）

5. 点击 **"Save"** 保存

---

### 第三步：等待 DNS 传播

DNS 配置完成后，需要等待几分钟到几小时让配置生效。

可以使用以下命令检查 DNS 是否生效：
```bash
nslookup tarot-yue.cn
```

或者访问 https://www.whatsmydns.net/ 检查全球 DNS 传播情况。

## 如果需要给后端也绑定自定义域名（可选）

### 第五步：添加 API 子域名 DNS 记录

1. 在 Cloudflare DNS 页面添加新记录：

   - Type: `CNAME`
   - Name: `api`
   - Target: `yang-production-c0f7.up.railway.app`
   - TTL: `Auto`
   - Proxy status: `Proxied`

2. 点击 "Save" 保存

3. 更新前端代码中的环境变量，将 [g:\1\1\.env.production](file:///g:/1/1/.env.production) 中的：
   ```
   VITE_API_URL=https://yang-production-c0f7.up.railway.app
   ```
   改为：
   ```
   VITE_API_URL=https://api.tarot-yue.cn
   ```

4. 提交并推送到 GitHub，Cloudflare Pages 会自动重新构建

## 注意事项

1. **DNS 传播时间**：DNS 配置可能需要几分钟到 48 小时才能完全生效
2. **HTTPS 证书**：Cloudflare 会自动为你的域名提供免费的 SSL 证书
3. **保持 Cloudflare 代理**：建议使用 "Proxied" 模式以获得更好的性能和安全性
